# [BE14] 명언 게시판(자바) 9~10단계 제출

## 출력 예시

```bash
== 명언 앱 ==
명령) 등록
명언 : 현재를 사랑하라.
작가 : 작자미상
1번 명언이 등록되었습니다.
명령) 등록
명언 : 과거에 집착하지 마라.
작가 : 작자미상
2번 명언이 등록되었습니다.
명령) 목록
번호 / 작가 / 명언
----------------------
2 / 작자미상 / 과거에 집착하지 마라.
1 / 작자미상 / 현재를 사랑하라.
명령) 삭제?id=1
1번 명언이 삭제되었습니다.
명령) 삭제?id=1
1번 명언은 존재하지 않습니다.
명령) 수정?id=2
명언(기존) : 과거에 집착하지 마라.
명언 : 현재와 자신을 사랑하라.
작가(기존) : 작자미상
작가 : 홍길동
명령) 목록
번호 / 작가 / 명언
----------------------
2 / 홍길동 / 현재와 자신을 사랑하라.
명령) 빌드
data.json 파일의 내용이 갱신되었습니다.
명령) 종료
```

## 구현 필요 명령 기능

* **종료** : 앱을 종료한다.
* **등록** : 명언을 등록한다. 번호는 1번부터 자동으로 매겨짐.
* **목록** : 삭제된 명언을 제외한 등록 명언을 출력.
* **삭제?id=X** : 등록되어 있는 명언을 지운다. 
    잘못된 아이디 지정에 따른 예외처리 필요.
    *(지운 번호는 영구적으로 사용 불가능)*
* **수정?id=X** : 등록되어 있는 명언을 수정한다.
    먼저 기존의 명언을 출력시킨 후,
    수정할 명언의 내용 및 작가를 입력해서 명언을 수정한다.
    잘못된 아이디 지정에 따른 예외처리 필요.
* **빌드** : `data.json` 파일에 지금까지 저장했던 명언 목록을 
    객체 형태로 저장한다.

## 추천 구조 정리(11단계 기준)

* `com.ll.wiseSaying.Main`: App 실행.
* `com.ll.wiseSaying.App`: 사용자의 입력을 받고, 
    그것이 WiseSayingController에게 넘겨야 하는지 판단,
    맞으면 메서드를 호출(인자와 함께).
* `com.ll.wiseSaying.WiseSayingController`:
    고객의 명령을 입력받고, 명령이 적절한지 응답을 표현.
* `com.ll.wiseSaying.WiseSayingService`:
    순수 비즈니스 로직. *(스캐너 및 출력 금지!)*
* `com.ll.wiseSaying.WiseSayingRepository`:
    데이터의 조회/수정/삭제/생성을 담당. *(스캐너 및 출력 금지!)*
* `com.ll.wiseSaying.WiseSaying`:
    명언 객체(번호/내용/작가)

## 데이터베이스 구조

* `./db/wiseSaying/lastId.txt` : 마지막으로 등록한 명언 번호
* `./db/wiseSaying/data.json` : 명언 객체 속성 저장 데이터베이스

## data.json 양식

```json
[
  {
    "id": 1,
    "content": "명언 1",
    "author": "작가 1"
  },
  {
    "id": 2,
    "content": "명언 2",
    "author": "작가 2"
  }
]
```
